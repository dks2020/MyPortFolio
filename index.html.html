01 / Power BI
Operations Performance Dashboard — ServCrust
Ops Analytics +
Business Problem

Leadership had no unified view of daily operations — order volumes, cancellations, service fulfillment rates, and exception handling were tracked across 6 disconnected spreadsheets. Decision-making was reactive, not proactive.


Data Sources

Order management system exports, MSME-wise performance logs, GMV transaction data, payout settlement records, non-accounting order flags.


Solution Approach

Built a unified star-schema data model in Power BI connecting all 6 sources. Designed 3-layer dashboard: Executive Summary → Team-Level Drill → Individual KPI. Implemented daily automated refresh and role-level security so each manager sees only their vertical.

Key DAX Logic
// YTD GMV with dynamic filter context YTD_GMV = CALCULATE( SUM(Orders[GMV]), DATESYTD(Calendar[Date]) ) // Cancellation Rate % Cancel_Rate% = DIVIDE( COUNTROWS(FILTER(Orders, Orders[Status]="Cancelled")), COUNTROWS(Orders), 0 ) // Pending Settlement Flag Pending_Days = DATEDIFF(Orders[SettlementDue], TODAY(), DAY)

Impact Delivered
↓ 70% reporting time ↑ 40% exception resolution speed CEO daily review enabled 6 reports → 1 dashboard
02 / Power BI + QMS
Quality Control Analytics — Error Tracking & Analyst Scoring
Quality Intelligence +
Business Problem

Quality errors were being tracked manually — audit results sat in email threads. No visibility into error trends by type, analyst, or process. Coaching was reactive with no data to back it up.


Data Sources

SharePoint-based QMS audit entries, analyst production logs, error taxonomy classification, team hierarchy mappings.


Solution Approach

Built the MERLIN QMS system — a SharePoint + Power BI integration capturing 100% of audits digitally. Created error Pareto analysis, analyst-level scorecards, and trend alerts. Enabled calibration sessions with visual proof points replacing anecdotal feedback.

Key DAX Logic
// QC Score per Analyst QC_Score = DIVIDE( SUMX(Audits, Audits[Points_Earned]), SUMX(Audits, Audits[Points_Possible]), 0 ) * 100 // Rolling 4-Week Error Trend Error_4W_Avg = CALCULATE( COUNTROWS(Audits), DATESINPERIOD(Calendar[Date], LASTDATE(Calendar[Date]), -28, DAY) )

Impact Delivered
↓ 35% repeat errors in 90 days 100% audit digitization ↓ 80% calibration prep time Analyst coaching data-driven
03 / Excel + Power BI
Employee Productivity & Stack Rank System — KFin Technologies
Performance Mgmt +
Business Problem

Performance reviews were subjective. No unified scoring across 100+ associates spanning multiple processes. Managers could not identify low-performers early or recognize high-performers fairly.


Data Sources

ATOM time tracking data, productivity volumes, QC scores, adherence logs, HR leave records, AHT from system timestamps.


Solution Approach

Designed a weighted Stack Rank algorithm combining 5 KPIs: productivity %, quality score, adherence, AHT efficiency, and behavioral ratings. Built automated weekly Power BI refresh with manager-view drill-through. System was later standardized across the Central Indexing division.

Key DAX Logic
// Weighted Stack Rank Score Stack_Score = ([Productivity_Score] * 0.35) + ([QC_Score] * 0.30) + ([AHT_Score] * 0.20) + ([Adherence_Score] * 0.15) // Relative Rank within Team Team_Rank = RANKX( ALLSELECTED(Employees[EmpID]), [Stack_Score], , DESC, Dense )

Impact Delivered
↑ 25% performance transparency Subjectivity eliminated Standardized across division 100+ associates tracked
04 / VBA + Power BI
ATOM — Automated Tracking & Operations Management System
Automation +
Business Problem

Work allocation was manual — supervisors spent 90+ minutes daily distributing tasks based on availability, priority, and skill. No time-tracking. No real-time queue visibility.


Data Sources

Inbound work queues, associate skill matrices, real-time availability flags, process priority rules, SLA timelines.


Solution Approach

Built ATOM — a VBA + MS Access system with dynamic work allocation engine using rule-based priority logic. Auto-assigns tasks to associates based on skill, availability, and queue age. Time tracking built in from login to log-off. Output feeds Power BI for intraday management visibility.

Technical Highlights
// Dynamic Allocation Logic (VBA pseudocode) For Each task In PendingQueue Set agent = GetAvailableAgent( skill := task.RequiredSkill, priority := task.SLAFlag ) If Not agent Is Nothing Then AssignTask(task, agent) LogTimestamp(agent, task, Now()) End If Next task

Impact Delivered
↓ 90 min/day supervisor overhead ↑ 30% throughput 100% time capture automated SLA breach alerts enabled
